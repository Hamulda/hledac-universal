## Bottlenecks

- **[HIGH]** `time.sleep` in async contexts — M1 crash risk (legacy async patterns)
- **[MED]** 26 coordinators in execution path — coordination overhead
- **[MED]** DuckDB writes on hot path — canonical storage may saturate
- **[LOW]** LMDB unchecked growth — no compaction on cleanup paths

---

## Recommendations

1. Replace blocking `time.sleep` with `asyncio.sleep` in async contexts
2. Profile coordinator dispatch overhead — consider batching advisory sidecars
3. Add LMDB size monitoring to `aggressive_cleanup()`
4. Verify DuckDB write batching for high-throughput finding ingestion
